변경 전...
  • body의 값을 읽고 수행하는 스크립트가 들어있기 때문에 파싱의 순서를 고려하여 넣어야한다
  • onload 함수를 이용하면 스크립트의 위치를 고려하지 않아도 된다. 하지만 onload가 들어간 함수가 로딩이 다 될때까지 그다음코드가 되지않는 효율적이지 않은 코드이다.
  • script 안에 async 혹은 defer 를 붙여주는것이 효율적
    • script async src="./script01.js"
    • script defer src="./script01.js"

    👍 결론적으로는 defer가 가장 효율적이다: 비동기 문서를 로드하고 html 파싱 후 모두 실행하기 때문.
